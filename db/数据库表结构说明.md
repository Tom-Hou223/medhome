# 家庭常备药品智能管理系统 - 数据库表结构说明

## 数据库信息
- **数据库名称**: `family_medicine_db`
- **字符集**: `utf8mb4`
- **排序规则**: `utf8mb4_unicode_ci`

---

## 数据表清单（共13张表）

### 1. 用户相关表（1张）

#### 1.1 users - 用户表
存储微信小程序用户的基本信息。

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | BIGINT | 用户ID | 主键 |
| openid | VARCHAR(100) | 微信openid | 唯一，非空 |
| phone | VARCHAR(20) | 手机号 | 唯一 |
| nickname | VARCHAR(100) | 昵称 | - |
| avatar_url | VARCHAR(500) | 头像URL | - |
| created_at | DATETIME | 创建时间 | 默认当前时间 |
| updated_at | DATETIME | 更新时间 | 自动更新 |

**索引**: `idx_users_openid`, `idx_users_phone`

---

### 2. 家庭相关表（4张）

#### 2.1 families - 家庭组表
存储家庭组的基本信息。

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | BIGINT | 家庭ID | 主键 |
| name | VARCHAR(100) | 家庭名称 | 非空 |
| creator_id | BIGINT | 创建者ID | 外键→users.id |
| invite_code | VARCHAR(20) | 邀请码 | 唯一 |
| invite_code_expires_at | DATETIME | 邀请码过期时间 | - |
| created_at | DATETIME | 创建时间 | 默认当前时间 |

**索引**: `idx_families_invite_code`

#### 2.2 family_user_roles - 家庭成员角色表
存储用户在家庭中的角色和关系。

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | BIGINT | 记录ID | 主键 |
| family_id | BIGINT | 家庭ID | 外键→families.id |
| user_id | BIGINT | 用户ID | 外键→users.id |
| role | ENUM | 角色：admin/member | 默认member |
| relationship | VARCHAR(50) | 家庭关系 | 默认member |
| joined_at | DATETIME | 加入时间 | 默认当前时间 |

**索引**: `unique_family_user`, `idx_family_user_roles_user`, `idx_family_user_roles_family`

#### 2.3 family_relationship_rules - 家庭关系规则表
定义家庭关系的规则和限制。

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 规则ID | 主键，自增 |
| relationship | VARCHAR(50) | 关系名称 | 唯一，非空 |
| max_count | INT | 最大数量 | NULL表示无限制 |
| description | VARCHAR(200) | 描述 | - |

**预置关系**: self, spouse, father, mother, son, daughter, grandfather_paternal, grandmother_paternal, grandfather_maternal, grandmother_maternal, other

#### 2.4 family_members - 家庭成员信息表

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | BIGINT | 成员ID | 主键 |
| family_id | BIGINT | 家庭ID | 外键→families.id |
| name | VARCHAR(100) | 成员姓名 | 非空 |
| relationship | VARCHAR(50) | 家庭关系 | - |
| age | INT | 年龄 | - |
| created_at | DATETIME | 创建时间 | 默认当前时间 |

## 触发器（6个）

1. **medicines_after_insert** - 药品插入后记录日志
2. **medicines_after_update** - 药品更新后记录日志
3. **medicines_after_delete** - 药品删除后记录日志
4. **plans_after_insert** - 计划插入后记录日志
5. **plans_after_update** - 计划更新后记录日志
6. **plans_after_delete** - 计划删除后记录日志
7. **family_members_after_insert** - 成员插入后记录日志
8. **family_members_after_update** - 成员更新后记录日志
9. **family_members_after_delete** - 成员删除后记录日志

---

## 存储过程（1个）

**clean_old_change_logs()** - 清理7天前的变更日志

---

## 定时事件（1个）

**clean_logs_daily** - 每天凌晨3点自动清理旧日志

## 注意事项

1. 所有涉及家庭数据的表都包含 `family_id` 字段，支持多家庭数据隔离
2. 使用触发器自动记录数据变更，支持多设备实时同步
3. 定时任务会自动清理7天前的变更日志，避免日志表过大
4. 所有外键都设置了 `ON DELETE CASCADE`，删除家庭时会自动清理相关数据
5. JSON 字段用于存储复杂数据结构（如 time_slots、recognition_result）
6. 使用 BIGINT 作为主键，支持雪花算法生成的分布式ID
