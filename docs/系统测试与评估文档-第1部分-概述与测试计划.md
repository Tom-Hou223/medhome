## 一、系统概述

### 1.1 项目背景
药效记是一款基于微信小程序的家庭常备药品智能管理系统，旨在解决家庭药品过期难发现、多成员用药混乱、老年人易漏服等实际问题。

### 1.2 技术架构
- **前端技术栈：** 微信小程序原生开发 + Vant Weapp UI组件库
- **后端技术栈：** Node.js + Express + MySQL
- **认证方式：** JWT Token + 微信登录
- **通信协议：** RESTful API + HTTPS

### 1.3 核心功能模块
1. 用户管理模块（登录、个人信息）
2. 家庭管理模块（家庭组、成员管理、权限控制）
3. 药品管理模块（录入、查询、编辑、删除）
4. 用药计划模块（创建、编辑、状态管理）
5. 用药记录模块（自动生成、状态更新）
6. 提醒通知模块（用药提醒、过期预警）
7. 数据统计模块（可视化图表）
8. 系统设置模块（老年模式、数据导出）

### 1.4 数据库设计
- **数据表数量：** 13张核心表
- **主要表：** users、families、medicines、plans、records
- **辅助表：** family_members、user_subscriptions、data_change_logs等
- **触发器：** 6个（用于数据变更日志）
- **存储过程：** 1个（清理旧日志）

## 二、测试目标与范围

### 2.1 测试目标
1. **功能完整性验证：** 确保所有功能模块按需求规格说明书正常运行
2. **性能指标达标：** 验证系统响应时间、并发处理能力符合要求
3. **安全性保障：** 检测系统安全漏洞，确保数据安全
4. **兼容性测试：** 验证不同设备、系统版本的兼容性
5. **用户体验评估：** 评估界面友好性、操作流畅性
6. **稳定性验证：** 长时间运行稳定性测试

- ✅ 提醒通知（用药提醒、过期预警、订阅管理）

## 三、测试策略与方法

### 3.1 测试策略

#### 3.1.1 测试阶段划分
1. **单元测试阶段（开发阶段）**
   - 开发人员自测
   - 代码审查
   - 关键函数单元测试

2. **集成测试阶段（第1-2天）**
   - 模块间接口测试
   - 数据流测试
   - API接口测试

3. **系统测试阶段（第3-6天）**
   - 功能测试
   - 性能测试
   - 安全测试
   - 兼容性测试

4. **验收测试阶段（第7-9天）**
   - 用户场景测试
   - 回归测试
   - 最终验收

#### 3.1.2 测试优先级
- **P0（最高优先级）：** 核心功能、数据安全、用户登录
- **P1（高优先级）：** 药品管理、用药计划、提醒通知
- **P2（中优先级）：** 数据统计、系统设置
- **P3（低优先级）：** 界面美化、辅助功能

### 3.2 测试方法

#### 3.2.1 黑盒测试
- **等价类划分法：** 输入数据分类测试
- **边界值分析法：** 测试边界条件
- **错误推测法：** 基于经验推测可能的错误
- **场景测试法：** 模拟真实用户使用场景

#### 3.2.2 白盒测试
- **语句覆盖：** 确保每行代码至少执行一次
- **分支覆盖：** 测试所有条件分支
- **路径覆盖：** 测试主要执行路径

#### 3.2.3 灰盒测试
- **接口测试：** 使用Postman测试API接口
- **数据库测试：** 验证数据一致性和完整性
- **日志分析：** 检查系统日志和错误日志
