# 药效记 - 家庭常备药品智能管理系统
## 需求规格说明书 v2.0

**编制日期：** 2026-03-09  
**项目名称：** 药效记（HomeMediRemind）

---

## 一、项目概述

### 1.1 背景与目标
本项目是基于微信小程序的家庭常备药品智能管理系统，解决药品过期难发现、多成员用药混乱、老年人易漏服等问题。

**核心目标：**
- 药品全生命周期管理
- 多成员家庭协同
- 智能提醒功能
- 老年用户适配

### 1.2 技术架构
- **前端：** 微信小程序 + Vant Weapp UI
- **后端：** Node.js + Express + MySQL
- **认证：** JWT Token
- **通信：** RESTful API

**2.1.2 个人信息管理**

### 2.2 家庭管理模块

**2.2.1 家庭组功能**
- 创建家庭（自动生成24小时有效邀请码）
- 加入家庭（输入邀请码）
- 切换当前家庭
- 退出/删除家庭

**2.2.2 成员管理**
- 查看成员列表（显示角色、关系）
- 修改成员角色（管理员/普通成员）
- 移除成员（仅管理员）
- 生成新邀请码

**2.2.3 成员信息管理**
- 添加成员详细信息（姓名、年龄、家庭关系）
- 用于用药计划关联
- 支持关系：父亲、母亲、儿子、女儿、爷爷、奶奶等

**2.2.4 权限控制**
- **管理员：** 可增删改药品、计划、成员
- **普通成员：** 可查看数据、确认用药

**核心接口：**
```
POST   /api/families/create
POST   /api/families/join
GET    /api/families/{id}/members
POST   /api/families/{id}/invite
PUT    /api/families/{id}/members/{userId}/role
DELETE /api/families/{id}/members/{userId}
```

### 2.3 药品管理模块

**2.3.1 药品录入（三种方式）**

**方式一：扫码识别**
- 扫描药品条形码
- 调用识别API获取药品信息
- 自动填充表单

**方式二：拍照识别**
- 拍摄药品包装
- OCR识别药品信息
- 自动填充表单

**方式三：手动录入**
- 直接填写表单

**必填字段：**
- 药品名称
- 过期日期

**可选字段：**
- 生产厂家、规格、分类、库存数量、单位、用法用量

**2.3.2 药品列表**
- 展示所有药品
- 搜索功能（按名称/厂家/分类）
- 状态标记：
  - 🟢 正常（>30天）
  - 🟡 即将过期（7-30天）
  - 🔴 已过期（≤0天）

**2.3.3 药品操作**
- 查看详情
- 编辑（仅管理员）
- 删除（仅管理员）

**核心接口：**
```
GET    /api/medicine/list
POST   /api/medicine/add
PUT    /api/medicine/update/{id}
DELETE /api/medicine/delete/{id}
POST   /api/medicine/recognize/barcode
POST   /api/medicine/recognize/image
```

### 2.4 用药计划模块

**2.4.1 创建用药计划**

**必填信息：**
- 药品名称（可从药品库选择或手动输入）
- 成员姓名（从成员列表选择）
- 开始日期
- 服药时间（可添加多个时间点，如：08:00, 12:00, 18:00）

**可选信息：**
- 结束日期（不填则长期有效）
- 频率（每日/每周/每月）
- 备注

**2.4.2 计划列表**
- 筛选：全部/进行中/已结束
- 显示：药品名、成员名、时间、状态
- 按成员分组显示

**2.4.3 计划状态管理**
- 自动判断：超过结束日期→已结束
- 手动操作：编辑、删除（仅管理员）

**核心接口：**
```
GET    /api/plan/list
POST   /api/plan/create
PUT    /api/plan/update/{id}
DELETE /api/plan/delete/{id}
```

### 2.5 用药记录模块

**2.5.1 记录自动生成**
- 根据用药计划每日自动生成记录
- 每个计划的每个时间点生成一条记录
- 初始状态：待服用（pending）

**2.5.2 记录状态**
- **待服用（pending）：** 未确认
- **已完成（completed）：** 已确认服药
- **已错过（missed）：** 标记为漏服

**2.5.3 记录操作**
- 确认服药（状态→已完成）
- 标记错过（状态→已错过）
- 按日期查询记录

**核心接口：**
```
GET  /api/records?date=YYYY-MM-DD
POST /api/records/add
POST /api/records/complete/{id}
POST /api/records/miss/{id}
```

## 三、数据库设计

### 3.1 核心数据表（共13张）

**1. users - 用户表**
```sql
id, openid, phone, nickname, avatar_url, created_at, updated_at
```

**2. families - 家庭表**
```sql
id, name, creator_id, invite_code, invite_code_expires_at, created_at
```

**3. family_user_roles - 家庭成员角色表**
```sql
id, family_id, user_id, role(admin/member), relationship, joined_at
```

**4. family_members - 家庭成员信息表**
```sql
id, family_id, name, relationship, age, created_at
```

**5. medicines - 药品表**
```sql
id, family_id, name, manufacturer, specification, category, 
stock, unit, expiry_date, dosage, status, created_at
```

**6. plans - 用药计划表**
```sql
id, family_id, medicine_name, member_name, frequency, 
time_slots(JSON), status, start_date, end_date, created_at
```

**7. records - 用药记录表**
```sql
id, plan_id, medicine_name, member_name, time, date, status, created_at
```

**8. user_subscriptions - 用户订阅表**
```sql
id, user_id, template_id, template_type, is_active, subscribed_at
```

**9. user_notification_settings - 用户通知设置表**
```sql
id, user_id, reminder_time, expiry_warning_days, created_at, updated_at
```

**10. recognition_history - 识别历史表**
```sql
id, user_id, family_id, type, input_data, recognition_result(JSON), is_added, created_at
```

**11. data_change_logs - 数据变更日志表**
```sql
id, family_id, table_name, record_id, action, user_id, changed_at
```

**12. family_relationship_rules - 家庭关系规则表**
```sql
id, relationship, max_count, description
```

### 3.2 数据关系
- 一个用户可属于多个家庭（多对多）
- 一个家庭有多个成员
- 药品、计划、记录都属于特定家庭（数据隔离）
- 一个计划生成多条记录（一对多）

### 3.3 触发器
- 药品/计划/成员的增删改自动记录到data_change_logs
- 用于多设备数据同步

## 七、业务流程

### 7.1 用户首次使用流程
```
微信登录 → 创建家庭 → 添加家庭成员信息 → 录入药品 
→ 创建用药计划 → 每日确认用药 → 查看统计
```

### 7.2 药品管理流程
```
选择录入方式（扫码/拍照/手动） → 识别/填写药品信息 
→ 确认保存 → 自动计算过期状态 → 临期自动预警
```

### 7.3 用药提醒流程
```
创建用药计划 → 系统自动生成每日记录 → 到达提醒时间 
→ 发送微信服务通知 → 用户打开小程序 → 确认服药 
→ 更新记录状态 → 更新日历标记
```

### 7.4 家庭协同流程
```
管理员创建家庭 → 生成邀请码 → 分享给家人 
→ 家人输入邀请码加入 → 管理员分配角色 
→ 共同管理药品和计划
```

## 八、部署方案

### 8.1 服务器配置（推荐）
- **CPU：** 2核
- **内存：** 4GB
- **硬盘：** 50GB SSD
- **带宽：** 5Mbps
- **操作系统：** Ubuntu 20.04 LTS

### 8.2 软件环境
- Node.js v16.x
- MySQL v8.0
- Nginx v1.18
- PM2（进程管理）

### 8.3 部署步骤
1. 安装Node.js、MySQL、Nginx
2. 创建数据库：`family_medicine_db`
3. 导入表结构：`mysql < database_migration.sql`
4. 配置数据库连接：修改`db.js`
5. 安装依赖：`npm install`
6. 启动后端：`pm2 start app.js`
7. 配置Nginx反向代理
8. 配置SSL证书（Let's Encrypt）
9. 微信小程序配置服务器域名

### 8.4 Nginx配置示例
```nginx
server {
    listen 443 ssl;
    server_name api.yourdomain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 十二、创新点总结

### 12.1 场景适配创新
覆盖"药品录入-分类存储-效期预警-用药提醒-过期处置"全流程，适配多成员家庭、老年用户、儿童用药等不同场景，支持多角色协同管理。

### 12.2 操作轻量化创新
基于微信小程序生态，设计"扫码录入+语音提醒+极简界面"操作流程，简化老年用户使用门槛，实现"即用即走"的轻量化体验。

### 12.3 提醒机制创新
采用"效期预警+用药提醒"双维度触发模式，支持自定义预警周期和提醒频率，通过微信服务通知确保提醒时效性，解决传统提醒易遗漏问题。

### 12.4 数据同步创新
通过数据变更日志表实现多设备实时同步，家庭成员在不同设备上的操作可以实时反映到其他成员设备。
