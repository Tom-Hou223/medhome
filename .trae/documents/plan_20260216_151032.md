# 修复年月选择器没有数据的问题

## 问题分析

用户反映年月选择器中只有2026年2月的数据，没有其他年份和月份的选项。

### 根本原因

* 在`index.js`的`initDate`函数中，`minDate`和`maxDate`被设置为只包含当前月份的范围
* 这导致年月选择器只能显示当前月份，无法选择其他年份和月份

## 解决方案

### 1. 修改日期范围设置

* 修改`initDate`函数，设置更宽的日期范围
* 例如：从当前年份的前5年到后5年
* 确保`minDate`和`maxDate`覆盖足够的年份范围

### 2. 保持功能兼容性

* 确保修改后的日期范围不会影响其他功能
* 保持日历的基本功能正常工作
* 确保选择器的显示和隐藏逻辑正确

## 技术实现

### 修改index.js中的initDate函数

```javascript
initDate: function() {
  const today = new Date();
  const year = today.getFullYear();
  const monthNumber = today.getMonth() + 1;
  const monthStr = String(monthNumber).padStart(2, '0');
  const dayStr = String(today.getDate()).padStart(2, '0');
  const selectedDate = `${year}-${monthStr}-${dayStr}`;
  const baseDate = today.getTime();

  // 设置更宽的日期范围：前5年到后5年
  const minYear = year - 5;
  const maxYear = year + 5;
  const minDate = new Date(minYear, 0, 1).getTime(); // 前5年1月1日
  const maxDate = new Date(maxYear, 11, 31).getTime(); // 后5年12月31日

  this.setData({
    currentDate: today,
    selectedDate: selectedDate,
    currentYear: year,
    currentMonth: monthNumber,
    calendarBaseDate: baseDate,
    minDate,
    maxDate
  });

  this.loadMonthStatus(year, monthNumber);
}
```

### 确保其他函数使用正确的日期范围

* 修改`onPrevMonth`、`onNextMonth`和`onYearMonthConfirm`函数
* 确保它们使用全局的`minDate`和`maxDate`，而不是重新设置为只包含当前月份的范围

## 预期效果

修改后，年月选择器将显示从当前年份前5年到后5年的所有年份和月份，用户可以自由选择不同的年月。

## 注意事项

* 确保修改后的日期范围不会导致性能问题
* 保持与现有功能的兼容性
* 确保选择器的显示效果正常
