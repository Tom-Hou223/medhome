# 修复年月选择器收不回去的问题

## 问题分析

用户反映年月选择器在选择日期后无法正常关闭（收不回去）。

### 可能原因

1. **事件绑定问题**：`bind:cancel` 或 `bind:confirm` 事件没有正确绑定

2. **数据更新问题**：`setData` 没有正确更新 `showYearMonthPicker` 的值

3. **组件交互问题**：`van-popup` 和 `van-datetime-picker` 组件的交互存在问题

4. **其他代码干扰**：可能有其他代码影响了 `showYearMonthPicker` 的值

## 解决方案

### 1. 检查事件绑定

* 确保 `van-datetime-picker` 的 `bind:confirm` 和 `bind:cancel` 事件正确绑定到对应的函数

* 确保 `van-popup` 的 `bind:close` 事件正确绑定到 `closeYearMonthPicker` 函数

### 2. 测试函数调用

* 在 `closeYearMonthPicker` 和 `onYearMonthConfirm` 函数中添加日志，测试函数是否被调用

* 验证函数调用时 `setData` 是否正确执行

### 3. 优化数据更新逻辑

* 确保 `showYearMonthPicker` 的值在所有需要关闭选择器的场景中都被正确设置为 `false`

* 检查是否有其他代码在后续操作中重新设置了 `showYearMonthPicker` 的值

### 4. 增强组件交互

* 为 `van-popup` 添加 `closeable` 属性，允许点击遮罩层关闭

* 确保 `van-datetime-picker` 的确认和取消按钮能够正确触发事件

## 技术实现

### 1. 修改 wxml 配置

```xml
<van-popup
  show="{{showYearMonthPicker}}"
  position="bottom"
  bind:close="closeYearMonthPicker"
  closeable="{{true}}"
  overlay="{{true}}"
>
  <van-datetime-picker
    type="year-month"
    value="{{calendarBaseDate}}"
    min-date="{{minDate}}"
    max-date="{{maxDate}}"
    bind:confirm="onYearMonthConfirm"
    bind:cancel="closeYearMonthPicker"
  />
</van-popup>
```

### 2. 增强 js 函数

```javascript
closeYearMonthPicker: function() {
  console.log('关闭年月选择器');
  this.setData({
    showYearMonthPicker: false
  });
},

onYearMonthConfirm: function(e) {
  console.log('确认年月选择');
  const value = e.detail;
  const date = new Date(value);
  const year = date.getFullYear();
  const month = date.getMonth() + 1;
  const monthFirstDate = new Date(year, month - 1, 1);
  const baseDate = monthFirstDate.getTime();
  const monthStr = String(month).padStart(2, '0');
  const selectedDate = `${year}-${monthStr}-01`;

  this.setData({
    currentYear: year,
    currentMonth: month,
    calendarBaseDate: baseDate,
    selectedDate,
    showYearMonthPicker: false
  });

  this.loadMonthStatus(year, month);
  this.loadData(selectedDate);
}
```

### 3. 确保数据一致性

* 检查是否有其他地方修改了 `showYearMonthPicker` 的值
* 确保在所有场景下都能正确关闭选择器

## 预期效果

修改后，年月选择器应该能够在以下场景下正常关闭：

1. 用户点击取消按钮
2. 用户点击确认按钮
3. 用户点击遮罩层

## 注意事项

* 确保修改不会影响其他功能
* 保持代码的可读性和可维护性
* 测试不同的操作场景，确保选择器在所有情况下都能正常关闭
